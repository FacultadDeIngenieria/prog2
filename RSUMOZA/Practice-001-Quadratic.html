1. Quadratic

You have to solve a Second grade equation of the form ax2 + bx + c.

For this create a class named Quadratic (in a new file) that get created from 3 Float numbers, each representing the coefficients a, b, and c.

This class have all this behaviour:

  • roots returns a list of numbers that represents the (x1 and x2).
    • If the equation has only one real solution then it should return one root.
    • If the equation has no real solutions then return an empty List.
  • toString should return a String representing the Quadratic as a String the form Y = A * X2 + B * X + C
  • derivation should return a String representing the derivative of the Quadratic function.

None of these methods receive any parameters, they should use the classes variables for their calculations and output.

Examples

To test your code, you can create a new class called QuadraticTest in a new file and test if the Quadratic methods work correctly, inside a main method.


    Quadratic q = new Quadratic(1,-5,6);
q.roots() should return [2,3]
q.toString() should return 1 * x^2 - 5 * x + 6
q.derivation() should return 2 * x - 5

2. Cars

You have to model a car. A car in our system is defined by the following data:

  • Brand
  • Model
  • Year of production
  • Kilometers Driven

Whenever a new Car gets created, it always starts with 0 kilometers

Also, the Car should be able to perform the following operations:

  • getDescriptiveName returns a String in upper camel case that indicates the year of production, brand and model of the car, e.g: "2019 Audi A4".
  • readOdometer returns a String that describes the kilometers driven, e.g: "This car has 23 kilometers on it"
  • updateOdometer receives a number representing the new amount of kilometers driven, this changes the current value that the Car has, only if this value is higher than the previous one.
  • incrementOdometer receives a number representing an amount in kilometers, this amount gets added to the current kilometers driven value of the Car.

3. Time

Create a class named Time with attributes for hours, minutes, and seconds. It should include:

  • Getters and setters for each attribute individually and all three together.
  • A method nextSecond() that increments the time by one second and returns a new Time object.
  • A method previousSecond() that decrements the time by one second and returns a new Time object.
  • A toString() method that returns the time in HH:MM:SS format.

Time Class Usage Example

Time time = new Time();
time.setHours(23);
time.setMinutes(59);
time.setSeconds(59);
System.out.println(time); // Output: 23:59:59

Time nextTime = time.nextSecond();
System.out.println(nextTime); // Output: 00:00:00

Time previousTime = nextTime.previousSecond();
System.out.println(previousTime); // Output: 23:59:59

9. Ball

Develop a Ball class that simulates bouncing within a 2D Box. It should include:

  • Attributes for the ball's position (x, y), radius, and deltas (xDelta, yDelta).
  • A method to move the ball based on its current position and deltas.
  • Methods to change the direction of the ball horizontally and vertically (to simulate bouncing).
  • A toString() method that shows the current state of the ball.
  • In a main method, try bouncing the ball within defined box boundaries.

Ball Class Usage Example

Ball ball = new Ball(50, 50, 5, 10, 10);
System.out.println(ball); // Initial state
// Simulate the ball movement
for (int i = 0; i < 5; i++) {
    ball.move();
    System.out.println(ball);
    // Assume conditions for bouncing off walls
    if (ball.getX() + ball.getXDelta() > 100 || ball.getX() + ball.getXDelta() < 0) {
        ball.reflectHorizontal();
    }
    if (ball.getY() + ball.getYDelta() > 100 || ball.getY() + ball.getYDelta() < 0) {
        ball.reflectVertical();
    }
}

10. Account

Model an Account class with id, name, and balance attributes. It should include:

  • Getters and setters for each attribute.
  • A method to credit (add) an amount to the account.
  • A method to debit (subtract) an amount from the account, with a check to ensure the balance is sufficient (if it is not, print a console message alerting this and do nothing).
  • A method to transfer an amount from the account into another Account, with a check to ensure the balance is sufficient (if it is not, print a console message alerting this and do nothing).

Account Class Usage Example

Account account = new Account("001", "John Doe", 1000.0);
System.out.println(account); // Output: Account[id=001,name=John Doe,balance=1000.0]

account.credit(500);
System.out.println("Balance after credit: " + account.getBalance()); // Output: Balance after credit: 1500.0

account.debit(200);
System.out.println("Balance after debit: " + account.getBalance()); // Output: Balance after debit: 1300.0

Account anotherAccount = new Account("002", "Jane Doe", 300.0);
account.transferTo(anotherAccount, 300);
System.out.println("Balance after transfer: " + account.getBalance()); // Output: Balance after transfer: 1000.0
System.out.println("Another account balance: " + anotherAccount.getBalance()); // Output: Another account balance: 600.0
